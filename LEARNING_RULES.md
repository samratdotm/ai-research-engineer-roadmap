# Learning Rules

## AI-assistance policy

AI tools are allowed and encouraged, but they cannot become a substitute for understanding.

1. Attempt a problem independently for at least 25 focused minutes.
2. Ask first for a hint, counterexample, or debugging question - not a finished solution.
3. Never accept code I cannot explain line by line.
4. Reimplement the core mechanism without AI autocomplete at least once per phase.
5. Add tests before trusting generated code.
6. Verify mathematical and API claims against primary sources.
7. Record meaningful AI assistance in the experiment report.
8. After using AI, close it and explain the solution from memory.

## Research integrity

- Never choose only the seed or run that looks best.
- Never change a grader between before/after results without reporting it.
- Never use the test set to make training decisions.
- Quarantine invalid runs; do not silently delete them.
- Preserve configs, dataset versions, model revisions, logs, and environment information.
- Report negative results and limitations.
- Distinguish observation, inference, and speculation in writing.

## Sustainable execution

- Ten core hours are the commitment. Extra time is optional.
- A missed week is rescheduled, not hidden.
- Every fourth week includes a lighter review or consolidation block.
- Sleep and work performance are not traded for stretch goals.
- Depth on the current phase beats starting another AI application.
