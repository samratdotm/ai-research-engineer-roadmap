# Phase 8 - Capstone and Interviews

## Capstone standard

The capstone must include:

- A consequential research question.
- At least two meaningful baselines.
- A fixed and documented evaluation protocol.
- Multiple runs or samples with uncertainty.
- At least one ablation.
- Failure and out-of-distribution analysis.
- Reproducible code and environment.
- A research-style report and concise demo.

## Interview tracks

### Python and coding

- Data structures, algorithms, clean Python, NumPy vectorization, debugging, and tests.

### ML fundamentals

- Metrics, optimization, generalization, model architecture, data problems, and experiment validity.

### Research

- Form a hypothesis, design an experiment, choose controls, analyze ambiguous results, and critique a paper.

### Systems

- Design a distributed eval platform, model-serving system, training pipeline, or safe agent workflow.

### Project deep dive

- Problem, constraints, alternatives, failure, experiment, evidence, limitation, and next research question.

## Exit test

Complete a mock loop containing one Python round, one ML fundamentals round, one experimental-design round, one system-design round, and one 45-minute project deep dive.
